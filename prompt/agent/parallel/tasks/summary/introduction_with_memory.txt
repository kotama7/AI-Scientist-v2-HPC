### Summary Memory Guidelines

When summarizing, record:
- **Core**: `iteration_number`, `success_status`, `key_metric`, `main_insight`
- **Archival**: Detailed findings, comparisons with previous iterations, lessons learned

Example:
```
<memory_update>
{
  "core": {"iteration_number": 5, "success_status": "improved", "key_metric": "0.912 accuracy"},
  "archival": [{"text": "SUMMARY iter 5: Achieved 0.912 accuracy (+0.03 from iter 4). Key changes: (1) increased batch size to 128, (2) added dropout 0.3. Convergence was stable, no overfitting observed. Loss curve smooth. Recommend trying learning rate decay next.", "tags": ["SUMMARY", "ITERATION_5", "IMPROVEMENT"]}]
}
</memory_update>
```

Before summarizing, retrieve context from previous iterations:
```
<memory_update>
{
  "archival_search": {"query": "previous iteration summary results", "k": 3}
}
</memory_update>
```
